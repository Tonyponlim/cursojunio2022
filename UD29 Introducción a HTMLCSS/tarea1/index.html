Secciones y líneas generales de un documento HTML5

La especificación html5 trae muchos elementos a los desarrolladores web, permitiéndoles describir la estructura de un documento web con la semántica estandarizada. Este documento describe estos elementos y como usarlos para definir el perfil de cualquier documento.

Problemas resueltos para html 5

La definición de la estructura de un documento en HTML 4 y su algorismo de perfilado es muy tosco y genera numerosos problemas:

  1. HTML5 quita la necesidad de elementos <div> para definir secciones semánticas sin definir valores específicos pra los atributos class, introduciendo un nuevo elemento, <secction> al elemento de sección HTML.
  2. Mezclar varios documentos es difícil, la inclusión de un sub-documento en un documento principal. Esto se resuelve en HTML5 con unos elementos de seccionados ( <article>, <section>, <nav> y <aside>) son simpre subsecciones de su sesión ancestra más cercana.
  3. HTML5 introduce el elemento <hgroup><h1>Justine></h1><h2>Les Malheurs de la vertu</h2></hgroup> crea el perfil 1. Justine).
